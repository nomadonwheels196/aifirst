# GTM (Go-To-Market)

## Purpose
Central hub for Sales, Marketing, and Customer Success operations. Aligns demand generation, pipeline creation, deal execution, onboarding, and renewals to drive ARR and retention (NRR/GRR).

## Scope
- Sales — pipeline stages, playbooks, case studies, collateral
- Marketing — campaigns, brand/content, website (AI_First_Site)
- Customer Success — onboarding, adoption, QBRs, renewals/expansion

## Current State
- Last Updated: 2025-11-07
- Status: active
- DRI: Sales Lead, Marketing Lead, CS Lead

## Key Metrics
- Top-of-funnel: MQLs, SQLs, CAC, Conversion Rates
- Sales: Pipeline Coverage, Win Rate, Sales Cycle Length, ASP
- Success: Onboarding Time-to-Value, GRR, NRR, Churn/Expansion

## Inputs
- ICP, personas, product roadmap, pricing/packaging
- CRM and web analytics data; campaign plans
- Brand guidelines and messaging

## Outputs
- Qualified pipeline, closed won revenue, published case studies
- Website and landing pages, content assets, campaigns
- Onboarding/checklists, QBR decks, renewal plans

## Directory Map
- `Sales/` — process, collateral, and customer stories
- `Marketing/` — campaigns and `AI_First_Site/` static website (see its README.md)
- `Customer_Success/` — playbooks, templates, runbooks

## Security & Privacy
- Treat any PII (prospects/customers) as sensitive. Store PII in approved systems (e.g., CRM); reference IDs here, don’t duplicate.
- Never commit credentials or tokens.
