# Customer Success

## Purpose
Ensure customers achieve outcomes: efficient onboarding, adoption, QBRs, renewals, and expansion to drive GRR/NRR.

## Suggested Structure
- `Onboarding/` — playbooks, checklists, templates
- `QBRs/` — deck templates, cadence, notes
- `Health_Scoring/` — rubric, thresholds, alerts
- `Renewals/` — timelines, playbooks, approval paths

## Current State
- Last Updated: 2025-11-07
- Status: active
- DRI: CS Lead

## Metrics
- Time-to-value, Adoption/Engagement, CSAT/NPS, GRR, NRR, Churn reasons

## Inputs
- Customer goals/use cases; product roadmap; support/escalation data

## Outputs
- Onboarding plans, QBR decks, renewal plans, expansion playbooks

## Security & Privacy
- Treat all customer data as sensitive. Use account IDs/links; avoid storing PII directly in this repo.

## Contributing
- Keep runbooks concise and role-based. Link to CRM/ticketing for live data.
